Structure of the Generated Code

This code was generated using the stdPojo Template. This template generates, an Interface(with Getter/setter methods for the Cobol fields) for each Cobol Record. This interface has 2 implementing classes:

  • One (JR) is implemented as a wrapper around a JRecord line class. This class has 2 methods toPojo and set to convert to/from the equivalent pojo class.

ReadDtar020 and WriteDtar020 programs

The ReadDtar020.java and WriteDtar020.java programs are sample read / write programs. They are not intended to be useful in them selves (they might not compile). They are intended to serve as a base from which you can build upon.

IDtar020Pojo Interface

The IDtar020Pojo Interface has get.../set... methods for all the Cobol fields in the Dtar020 Record Definition. The Interface IDtar020Pojo is implemented by 2 classes:

  • LineDtar020JR.java is Wrapper for a standard JRecord Line with the appropriate Getter/setter methods. Behind the scenes it call the line.getFieldValue(..) ... methods to get/retrieve the values. There are the 2 other Methods for converting to/from a LineDtar020Pojo:
    • toPojo() method which converts this line into a LineDtar020Pojo class.
    • set(IDtar020 value) which will update the line with the values from a LineDtar020Pojo line.
  • LineDtar020Pojo.java implments the Interface using standard java types. It does not use or reference JRecord.

The IDtar020Pojo Interface extends 2 other interfaces:

  • The IDtar020 Interface defines the get... methods.
  • The IDtar020Upd Interface defines the set... methods.

Finally there is the Code.java class that has methods to copy between to classes that implement the IDtar020Pojo interface.
